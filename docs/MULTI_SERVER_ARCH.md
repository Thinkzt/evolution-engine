## 节点通信协议

### RPC框架选择

| 方案 | Stars | 特点 | 推荐度 |
|------|-------|------|--------|
| **gRPC** | - | 高性能/多语言 | ⭐⭐⭐⭐⭐ |
| **Apache Thrift** | - | Facebook出品 | ⭐⭐⭐⭐ |
| **JSON-RPC** | - | 简单/通用 | ⭐⭐⭐ |

### 通信流程

```
1. 任务注册 → 天枢接收请求
2. 任务分发 → 天权分析负载
3. 任务执行 → 分配到对应节点
4. 结果回收 → 汇总返回用户
5. 状态同步 → 各节点心跳保活
```

## 故障转移机制

### 心跳检测

```python
class HeartbeatMonitor:
    """每5秒检测节点存活"""
    interval = 5
    
    def check(self, node):
        """连续3次失败=节点故障"""
        if node.failures >= 3:
            self.trigger_failover(node)
```

### 自动切换

```
节点故障 → 检测 → 标记为down
         → 转移 → 任务分发到健康节点
         → 告警 → 通知管理员
         → 恢复 → 节点修复后自动加入
```
